编译php并与nginx整合

1. 安装mysql

yum install mysql mysql-devel 客户
yum install mysql-server服务
yum install dg dg-devel

2. 安装php

安装php需要注意的是：

  • 如果原来与apache编译过的php需要重新编译；

  • apache一般是把php当做自己的一个模块来启动的.而nginx则是把http请求变量(如 get,user_agent等)转发给 php进程,即php独立进程,与nginx进行通信. 称为 fastcgi运行方式，是不能用于nginx的.需要进入目录执行make clean重新执行编译。

要考虑需要编译的功能、编译在哪

支持mysql、gd库要支持、字体要支持、以fpm(fascgi)方式运行，那么就有了mysql,gd,ttf,fpm

不知道怎么添加编译可以查看帮助命令：

./configure -help | grep mysql

字体需要安装freetype：

yum install freetype

那么编译就可以这么写：

./configure --prefix=/usr/local/php \
--with-mysql=mysqlnd \
--enable-mysqlnd \
--with-gd \
--enable-gd-native-ttf \
--enable-gd-jis-conv
--enable-fpm

如果编译过程中遇到错误就看看提示，一般都是少编译了什么然后再加上重新编译然后执行:

make && make install

复制php配置文件,进入到php目录

cp /usr/local/src/php-5.6.17/php.ini-development ./lib/php.ini

复制php进程管理器php-fpm配置文件

cp /etc/php-fpm.conf.default etc/php-fpm.conf

运行php_fpm

/sbin/php-fpm

3. 简单配置

记住核心的一句话

发请求信息给php进程默认端口9000，让php进程来处理，
指定该目录下的php文件，包含fastccgi_params参数

location ~ \.php$ {
  root html;
  fastcgi_pass 127.0.0.1:9000;
  fastcgi_index index.php;
  fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
  include fastcgi_params;
  }

区别于apache+php：

./configure --prefix=/usr/local/php \
--with-mysql=mysqlnd \
--enable-mysqlnd \
--with-gd \
--enable-gd-native-ttf \
--enable-gd-jis-conv
--with-apxs2=/usr/local/httpd/bin/apxs

遇到问题：

configure: error: xml2-config not found. Please check your&nb

解决办法参考：

http://blog.sina.com.cn/s/blog_7253980a0101gw3n.html

安装ecshop

1. 下载ecshop

下载ecshop，把upload目录传到

/usr/local/nginx/html/

更改ecshop配置文件：ecshop/includes/init.php

error_reporting(0);
@ini_set('display_errors',        0);

删除：

if ((DEBUG_MODE & 1) == 1) {
  error_reporting(E_ALL);
} else {
    error_reporting(E_ALL ^ (E_NOTICE | E_WARNING));
}
if ((DEBUG_MODE & 4) == 4) {
    include(ROOT_PATH . 'includes/lib.debug.php');
}

在ecshop/install/inclodes/init.php中更改：

@ini_set('display_errors', 0);
error_reporting(0);

还有/usr/local/nginx/html/ecshop/admin/includes/init.php中同样也要修改

2. 配置nginx.conf

默认寻找文件index.php

3. 测试步骤

访问ecshop提示安装-》更改权限-》连接数据库

4. 一般会遇到的问题：

windows上传到liunx上的文件一般都是权限错误和大小写问题

chmod o+w temp
chmod o+w themes
chmod o+w cert -R
chmod o+w images -R
chmod o+w data -R
chmod o+w temp -R
chmod o+w themes -R

连接mysql报错：

root@localhost lib]# mysql -uroot -p
Enter password:
ERROR 2002 (HY000): Can't connect to local MySQL server through socket                                         '/var/lib/mysql/mysql.sock' (2)

解决方案：

检查mysql状态

/etc/init.d/mysqld status;

脚本启动mysql:

/etc/inint.d/mysqld start

安装时候报错：

原因：（本地localhost连接走sock,可以直接输入127.0.0.1）

解决方法：

①修改/usr/local/httpd/lib/php.ini中

; Default socket name for local MySQL connects.  If empty, uses the built-in
; MySQL defaults.
; http://php.net/mysql.default-socket
 mysql.default_socket = /var/lib/mysql/mysql.sock

② 强制关掉php-fpm进程

pkill -9 php-fpm

③ 重启fpm

/usr/local/httpd/sbin/php-fpm

tips:

查看具体的详细参数

mysql -uroot -p -e "show status\G;"

gzip+nginx压缩提升网速

1. 原理

浏览器-》请求-》声明可以接受的压缩（gzip defate compress sdch[google支持]）服务器-》回应-》把内容用gzip方式压缩-》发给浏览器浏览《-解码gzip《接收gzip压缩内容《

2. 配置常用参数

gzip on|off;               #是否开启gzip
gzip_buffers 32 4K| 16 8K  #缓冲(压缩在内存中缓冲几块? 每块多大?)
gzip_comp_level [1-9]      #推荐6 压缩级别(级别越高,压的越小,越浪费CPU资源)
gzip_disable               #正则匹配UA 什么样的Uri不进行gzip
gzip_min_length 200        # 开始压缩的最小长度(再小就不要压缩了,意义不在)
gzip_http_version 1.0|1.1  # 开始压缩的http协议版本(可以不设置,目前几乎全是1.1协议)
gzip_proxied               # 设置请求者代理服务器,该如何缓存内容
gzip_types text/plain application/xml # 对哪些类型的文件用压缩 如txt,xml,html ,css（html类型不写也会压缩）（可参考/usr/local/nginx/conf/mime.types）
gzip_vary on|off           # 是否传输gzip压缩标志

要注意：

①Transfer-Encoding: hunked代表分块传输，看不出长度

②压缩长度不易太小

③图片/MP3等二进制文件不必压缩（省不了多少还浪费资源）

nginx设置缓存设置

1. 作用：

提高网站性能，提升负载

2. 应用场景

新闻之类的一旦发布就很少改动的网站

3. 设置

用nginx的expires设置

expires 1d;

如果http请求不变浏览器就会把之前请求下来的资源拿来用，此时查看请求信息可以看到是304 和expires
