nginx简单配置管理(2)

发表于 2016-03-14   |   分类于 linux   |  

nginx 信号量，nginx假单配置，

nginx日志管理，按时间转存日志

一、nginx 信号量

开启，关闭，重启

kill -INT 26661
     -TERM (紧急杀掉)
     -QUIT请求结束后在关闭
     -HUP改配置文件，平滑的重读配置文件
     -USR1重读日志，
     -USR2 升级用
     -WINCH优雅关闭旧程序配合usr1使用

eg:

kill -HUP `cat logs/nginx.pid`

也可以也哦那个命令来管理：

./sbin/nginx -s reload stop

二、nginx 简单配置

nginx虚拟主机的配置

打开配置文件nginx.conf

全局区

worker_processes 1;       工作的子进程数量,可以修改,但太大无益,因为要争夺CPU,一般设置为 CPU数*核数

配置nginx连接的特性，如1个word能同时允许多少连接

Event {
   worker_connections  1024; // 这是指 一个子进程最大允许连1024个连接
}

配置http服务器的主要段

http {
  Server1 { // 虚拟主机段
        Location {  //定位,把特殊的路径或文件再次定位 ,如image目录单独处理
        }             /// 如.php单独处理
 }

 Server2 {
 }
}
简单的配置：

基于域名的虚拟主机

server {
    listen 80;
    server_name z.com;
    location / {
      root z.com;
      index.html;
    }
}

软重启nginx读取配置文件:

/usr/local/nginx/sbin/nginx -s reload

或者

kill -HUP `cat logs/nginx.pid`

然后解析host,访问z.com就可以访问了

基于端口的虚拟主机配置
server {
    listen 8080;
    server_name 192.168.1.204;

    location / {
            root /var/www/html8080;
            index index.html;
    }
}

三、nginx 简单日志管理

打开配置文件会看到

nginx的server段

#access_log  logs/host.access.log  main;（访问日志的文件是logs/host.access.log 使用main格式，也可以自定义格式）

main格式（是我们定义好一种日志的格式,并起个名字,便于引用 ）：

log_format  main  '$remote_addr（远程IP ） - $remote_user [$time_local]（远程用户/用户时间 ） "$request"（请求方法(如GET/POST) ） '
#                 '$status $body_bytes_sent （请求体body长度 ）"$http_referer（referer来源信息 ）" '
#                 '"$http_user_agent（用户代理/蜘蛛 ,被转发的请求的原始IP ）" "$http_x_forwarded_for（在经过代理时,代理把你的本来IP加在此头信息中,传输你的原始IP ）"';

声明一个独特的log_format并命名（选项可以自由选择）

log_format  mylog '$remote_addr- "$request" '
                  '$status $body_bytes_sent "$http_referer" '
                  '"$http_user_agent" "$http_x_forwarded_for"';

针对不同的server做不同的Log ,(有的web服务器不支持,如lighttp)

access_log logs/access_8080.log mylog;
声明log           log位置                  log格式;

四、按时间转存日志

1, 日志的定时切割，shell脚本，定时

date -s '2015-12-28 17:32'     修改系统时间
clock -w                       修改后写入日志

date -d yesterday              查看昨天时间
date -d yesterday +%Y%m%d      格式化时间

注意：安装ntpdate时间服务器，更新系统时间

2, 写脚本

命名runlog.sh

#!/bin/bash
LOGPATH=/usr/local/nginx/logs/access.log (日志路径)
BASEPATH=/data (将要保存路径)


bak=$BASEPATH/$(date "+%Y-%m-%d-%H-%M-%S").access.log（拼写命名路径）
#echo $bak

mv $LOGPATH $bak （把日志从原来路径移到拼写路径）
touch $LOGPATH （写入新的日志名）

kill -USR1 `cat /usr/local/nginx/logs/nginx.pid`（重写日志）

3, 让脚本生效

执行crontab -e
写入内容(分 时 日 月 周)
*/1 * * * * sh /data/runlog.sh

4， 如果想按月份创建目录按天切割管理：

#!/bin/bash

LOGPATH=/usr/local/nginx/logs/host.access.log

BASEPATH=/data/$(date "+%Y%m")

mkdir -p $BASEPATH

bak=$BASEPATH/$(date "+%d%H%M").host.access.log
#echo $bak

mv $LOGPATH $bak
touch $LOGPATH

kill -USR1 `cat /usr/local/nginx/logs/nginx.pid`
